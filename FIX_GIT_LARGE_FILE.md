# Решение проблемы с большим GIF файлом

## Проблема
GitHub отклоняет push, потому что в истории Git есть старый файл `bandicam-2023-10-01-13-16-43-801.gif` размером 153.55 MB (превышает лимит 100 MB).

## Решение

### Вариант 1: Удалить файл из Git истории (рекомендуется)

Выполните следующие команды в терминале:

```bash
# 1. Удалить файл из Git индекса (но оставить локально)
git rm --cached "demo/bandicam-2023-10-01-13-16-43-801.gif"

# 2. Добавить новый файл с пробелами (если еще не добавлен)
git add "demo/bandicam 2023-10-01 13-16-43-801.gif"

# 3. Закоммитить изменения
git commit -m "Заменить большой GIF на оптимизированную версию"

# 4. Если файл уже был запушен ранее, нужно удалить его из истории
git filter-branch --force --index-filter "git rm --cached --ignore-unmatch 'demo/bandicam-2023-10-01-13-16-43-801.gif'" --prune-empty --tag-name-filter cat -- --all

# 5. Принудительно запушить изменения
git push origin --force --all
```

### Вариант 2: Использовать Git LFS (если нужно сохранить большой файл)

Если вам действительно нужен большой файл:

```bash
# 1. Установить Git LFS (если еще не установлен)
# Windows: choco install git-lfs
# Mac: brew install git-lfs

# 2. Инициализировать Git LFS
git lfs install

# 3. Отслеживать GIF файлы через LFS
git lfs track "*.gif"

# 4. Добавить .gitattributes
git add .gitattributes

# 5. Добавить файлы и закоммитить
git add .
git commit -m "Использовать Git LFS для GIF файлов"
git push
```

### Вариант 3: Простое решение (если старый файл не был запушен)

Если старый файл еще не был запушен в удаленный репозиторий:

```bash
# 1. Удалить файл из индекса
git rm --cached "demo/bandicam-2023-10-01-13-16-43-801.gif"

# 2. Убедиться, что новый файл добавлен
git add "demo/bandicam 2023-10-01 13-16-43-801.gif"

# 3. Закоммитить
git commit -m "Заменить большой GIF на оптимизированную версию"

# 4. Запушить
git push
```

## Проверка

После выполнения команд проверьте:

```bash
# Проверить размер файлов в Git
git ls-files -s demo/ | grep gif

# Проверить статус
git status
```

## Рекомендации

1. **Оптимизируйте GIF** - используйте онлайн инструменты для уменьшения размера:
   - https://ezgif.com/optimize
   - https://compressor.io/

2. **Используйте .gitignore** - добавьте большие файлы в `.gitignore`:
   ```
   # Большие медиа файлы
   *.mp4
   *.mov
   demo/*.mp4
   ```

3. **Рассмотрите альтернативы**:
   - Загрузите GIF на внешний хостинг (Imgur, Giphy)
   - Используйте ссылку на YouTube видео
   - Создайте более короткий GIF (10-20 секунд)
